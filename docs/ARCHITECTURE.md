# 架构说明

## 当前产品边界

Codex2API 当前是一个 **账号管理后台系统**，不再对外提供公共模型中转接口。

## 核心模块

### 1. 前端管理台

目录：`frontend/`

负责：
- 账号列表与导入
- 代理管理
- 系统设置
- 运维面板
- CPA Sync 页面

### 2. 管理 API

目录：`admin/`

核心文件：`admin/handler.go`

负责：
- 账号 CRUD / 导入 / 导出 / 迁移
- 刷新状态 / 刷新 AT / 测试连接
- 代理增删改查与测试
- 系统设置
- 运维数据
- CPA Sync 控制

### 3. 账号调度与刷新

目录：`auth/`

核心文件：
- `auth/store.go`
- `auth/proxy_manager.go`

负责：
- 内存账号池
- 启动后的初始化与后台刷新
- 用量探测与恢复探测
- 动态代理分配
- 代理失效切换
- 代理轮换

### 4. 数据层

目录：`database/`

负责持久化：
- 账号
- 系统设置
- 代理池
- 账号事件与相关状态

### 5. 缓存层

目录：`cache/`

用于：
- token/session 类缓存
- 运行期辅助状态

## 请求流向

### 管理台请求

`Browser -> Gin Router -> admin.Handler -> auth.Store / database / cache`

### 后台刷新任务

`auth.Store -> 代理解析 -> 上游网络请求 -> 状态/用量回写`

## 代理优先级

当前代理解析优先级：

1. 文件中自带代理信息
2. 动态代理
3. 代理池
4. 全局代理

当已有代理不可用时，允许按规则回退到动态代理重新获取新代理。
