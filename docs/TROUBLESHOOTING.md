# 故障排查

## 1. 管理台打不开

检查：

```bash
curl -i http://localhost:8080/health
```

如果健康检查正常但 `/admin/` 打不开，优先确认：
- 前端是否已执行 `npm run build`
- `frontend/dist` 是否存在
- 反向代理是否正确转发 `/admin/`

## 2. 登录后接口 401

检查 `ADMIN_SECRET` 或数据库中的管理密钥配置，确认前端请求头是否正确携带：

```http
X-Admin-Key: <ADMIN_SECRET>
```

## 3. 导入账号后状态不刷新

优先检查：
- 后端启动日志是否正常
- 账号是否已成功入库
- 后台刷新任务是否启动
- 上游网络是否可达
- 代理配置是否有效

## 4. 动态代理没有生效

检查：
- 系统设置中的动态代理 URL 是否可访问
- 返回数据是否包含 `ip` 和 `port`
- 账号当前代理模式是否允许动态代理
- 是否被文件内代理或代理池优先命中

## 5. 代理测试失败或使用中断网

检查：
- 代理池可用性
- 动态代理提供方是否正常
- 全局代理是否可用
- 是否触发了代理自动切换

如果是运行过程中断网，应确认账号是否已切换到新代理。

## 6. 刷新状态 / 测试连接过慢

检查：
- `max_concurrency`
- `global_rpm`
- 代理网络延迟
- 上游接口响应时间
- 账号数量是否显著增长

## 7. CPA Sync 不工作

检查：
- CPA 相关设置是否完整
- Mihomo 配置是否正确
- 同步间隔是否过短
- 后端日志中是否有同步错误

## 8. 建议收集的信息

排查问题时建议同时提供：
- `/health` 返回内容
- 浏览器控制台报错
- 后端日志
- 具体账号是否配置代理/动态代理
- 问题发生的具体操作（导入、刷新、测试、更新 AT 等）
